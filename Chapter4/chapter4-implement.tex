\section{Data Import and Cleaning}
The dataset is imported from Excel into MATLAB. After importing, the data is checked for missing values, invalid entries and negative values. Rows containing invalid data are removed. Total energy is calculated by adding grid energy and DG energy.

The cleaned dataset is then used for KPI calculation and modeling. This step is important because incorrect or missing data can reduce prediction accuracy and create wrong simulation outputs.

The preprocessing workflow is:
\begin{enumerate}
    \item Excel Dataset
    \item MATLAB Import
    \item Data Cleaning
    \item Feature Creation
    \item KPI Calculation
    \item Model Training
\end{enumerate}

\section{MATLAB-Based Digital Twin Implementation}
The MATLAB implementation includes KPI calculation, regression modeling, prediction and scenario simulation. The linear and quadratic models are trained using production and total energy data. The quadratic model is selected because it gives better performance.

The digital twin takes production as input and predicts energy consumption. It then calculates cost, efficiency, EPU and emissions. The digital twin can also simulate changes such as DG reduction, production increase and base-load reduction.

The general working is:
\begin{enumerate}
    \item Input Production
    \item Predict Energy
    \item Calculate KPIs
    \item Simulate Scenario
    \item Display Result
\end{enumerate}

\section{AI Forecasting, Load Scheduling and Anomaly Detection}
The AI forecasting system uses LSTM to predict next-day energy demand. The forecast is compared with the safe grid limit. If the predicted energy is below the grid limit, the system gives a safe status. If predicted demand exceeds the grid limit, the system recommends load shifting.

The load scheduling logic divides the day into time blocks such as night, early morning, morning peak, afternoon and evening. If high demand is predicted, heavy loads are shifted away from peak periods to avoid DG usage.

Anomaly detection is performed using a 7-day rolling mean and standard deviation. If a day's energy consumption deviates more than 2 sigma from the rolling average, it is flagged as abnormal. This helps identify unusual energy usage caused by machine faults, maintenance or sudden production changes.

\section{Dashboard Prototype Implementation}
The dashboard prototype displays the output of the digital twin in a user-friendly format. It helps factory managers understand the current energy condition and make better decisions.

The dashboard includes:
\begin{enumerate}
    \item Total energy card.
    \item Cost card.
    \item CO2 emission card.
    \item DG usage card.
    \item Energy trend graph.
    \item AI forecast result.
    \item Load-balancing result.
    \item What-if simulator.
    \item Anomaly detection chart.
    \item Schedule advisor.
\end{enumerate}
